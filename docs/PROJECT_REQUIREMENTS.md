## 🚀 CI/CD 구성

### 플랫폼: GitHub Actions

- 무료 티어로 PR별 테스트
- 공개 리포에 이상적

### 필수 단계

#### 1. 자동 테스트 실행

- 커밋/PR 시 자동 실행
- 실패 시 머지 차단
- E2E 테스트 포함

#### 2. 코드 품질 검사

- ESLint 린트 검사
- Prettier 포맷 검증
- TypeScript 타입 체크
- 커버리지 검증 (70%+ 목표)

### CI/CD 워크플로우

```yaml
트리거: PR 생성/업데이트, main 브랜치 push
1. 린트 검사 (ESLint, Prettier)
2. 타입 체크 (TypeScript)
3. 유닛/통합 테스트 (선택적)
4. E2E 테스트 (Playwright)
5. 커버리지 리포트
6. 배포 (Vercel/기타 플랫폼)
```
